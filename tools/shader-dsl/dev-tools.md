# Shader DSL Developer Tooling

Complete toolkit for shader DSL development created during v2 DSL implementation.

## Tools Created

### 1. test-dsl.clj - Pure Clojure DSL Validator ✅

Tests DSL macro compilation without browser. Catches 90% of bugs before ClojureScript.

**Usage:**
```bash
clojure -M .claude/scripts/test-dsl.clj
```

**Tests:**
- Simple if-block
- if-block with let-block branches
- var-block with assign
- do flattening nested let
- Complex Bezier pattern

**Output:** AST structure verification

### 2. validate-wgsl.sh - Naga WGSL Validator ✅

Validates WGSL syntax using naga (Rust validator from wgpu team).

**Usage:**
```bash
# File input
./validate-wgsl.sh shader.wgsl

# Stdin input  
echo "$wgsl" | ./validate-wgsl.sh
```

**Requires:** cargo install naga-cli

### 3. test-shader-wgsl.sh - End-to-End Test ✅

Tests the complete generated WGSL from quadratic Bezier v2.

**Usage:**
```bash
./test-shader-wgsl.sh
```

**Validates:** The actual WGSL generated by v2 DSL

## Debugging Workflow

### Fast Iteration (No Browser)

```bash
# 1. Test DSL pattern
clojure -M .claude/scripts/test-dsl.clj

# 2. If AST looks good, compile in browser
npx shadow-cljs watch app

# 3. Check generated WGSL in console
# Browser: console.log(namespace.shader_var)
```

### WGSL Syntax Verification

```bash
# Extract WGSL from browser console log
# Copy to file: /tmp/shader.wgsl
./validate-wgsl.sh /tmp/shader.wgsl
```

### Root Cause Analysis

When debugging complex failures:

1. **Test DSL compilation** (test-dsl.clj)
   - Verifies macro expansion
   - Shows AST structure
   
2. **Check shadow-cljs logs** (logs/shadow-*.log)
   - Compilation errors
   - Type mismatches
   
3. **Examine browser console**
   - Runtime codegen errors
   - WGSL parse errors
   
4. **Validate generated WGSL** (naga)
   - Syntax correctness
   - Type checking

## Best Practices Learned

### 1. Test Macros in Isolation

```bash
# Create test case
cat > /tmp/test-pattern.clj << 'EOF'
(require '[nirvikalpa.shader.dsl :as dsl])
(def result (dsl/compile-body ['(your-pattern-here)]))
(println "Result:" result)
EOF

clojure -M /tmp/test-pattern.clj
```

### 2. Validate AST Structure

Look for:
- Keywords (`:if-block`) not symbols (`if-block`)
- Flat arrays not nested `[[:do [...]]]` vs `[:do ...]`
- Strings for variables not symbols

### 3. Use REPL for Quick Tests

```clojure
(require '[nirvikalpa.shader.dsl :as dsl])

;; Test specific pattern
(dsl/compile-simple-expr '(if-block (> x 0.5) a b))
;; => [:if-block [:> "x" 0.5] "a" "b"]
```
